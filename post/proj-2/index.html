Problem 7: 10001st prime (420,695)

Problem: By listing the first six prime numbers: 2, 3, 5, 7, 11, and 13, we can see that the 6th prime is 13.
What is the 10,001st prime number?

Analysis: The task asks for gathering 10,001 prime numbers. Can start by building a list of primes and display the 10001st number when we reach that length.

Solution: We know the first prime number is 2, so we can start the list from it. For every natural number, we need to check if it is a prime. A trick here is that if a number has a factor pair, then one of the factor must be less than or equal to the square root. So we can test whether the number is divisible from 2 to only up to the square root of that number. This trick will reduce workloads and make the function runs faster. Then form a list of prime numbers with the length = n, so that the nth number in the list is our desired result.

Function:

def getprime(n):
    primes = []
    x = 2

    def is_prime(num):
        '''inner function to check if the number is prime'''
        for i in range(2, int(num ** 0.5) + 1):
            if num%i == 0:
                return False
        return True

    while len(primes)!= n:
        if is_prime(x):
            primes.append(x)
        x+=1
    return primes[-1]

Answer: By running the function getprime(10001), the 10,001st prime number is 104743.

Problem 26 Reciprocal cycles (82,622)

Problem: A unit fraction contains 1 in the numerator. The decimal representation of the unit fractions with denominators 2 to 10 are given:

1/2 = 0.5
1/3 = 0.(3)
1/4 = 0.25
1/5 = 0.2
1/6 = 0.1(6)
1/7 = 0.(142857)
1/8 = 0.125
1/9 = 0.(1)
1/10 = 0.1

Where 0.1(6) means 0.166666…, and has a 1-digit recurring cycle. It can be seen that 1/7 has a 6-digit recurring cycle.

Find the value of d < 1000 for which 1/d contains the longest recurring cycle in its decimal fraction part.

Analysis: For this problem we can only check the prime number below the limit according to Fermat’s little theorem that 1/d has a cycle of n digits if 10^n mod d = 1 for prime d. That d is the full reptend prime. The solution is the first full reptend prime below the limit.

Solution:

  • First I need to find all prime numbers below the limit by using Eratosthenes algorithm. I create a list primes to store n elements that each has the initial values 1 to identify the number at that index is a prime. I set 0 and 1 index as 0 since we know they are not primes. Then apply Eratosthenes algorithm to determine if the ith number is assigned as 0 (not prime). Exclude all multiples of prime numers that no greater than the square root of n. (For numbers that already assigned as 0 before the ith number, don’t need to check their multiples again.) Then uses extended slicing to assign 0’s for the excluded numbers.
    • For example, when i=5, 2*5=10, so 10 is not a prime that need to be assign as 0. But since 2 is the minimum factor of 10, 10 is already assigned as 0 when i=2, so no need to check it again.
  • After finish checking, return the list of indicies that are assigned to be 1, which identify the prime numbers. That list is all prime numbers below the limit.
  • Check full reptend prime from the last one in the prime number list. And return the first full reptend prime below the limit as our answer.

Function:

def recuring_cyc(n):

    def find_primes(n):

        '''inner function to find all primes in
        first n natural numbers '''

        primes = [1]*n
        primes[0] = primes[1] = 0
        for i in range(2, int(n**0.5)+1):
            if primes[i]:
                primes[i*i:n:i] = [0] * ((n-1-i*i)//i+1)
        return [i for i, x in enumerate(primes) if x == 1]

    primes = find_primes(n)
    for d in primes[::-1]:
        p = 1
        while (10 ** p) % d != 1:
            p+=1
        if d-1 == p:
            break
    return d

Answer: By running the function recuring_cyc(1000), the longest recurring cycle for 1/d where d < 1000 is when d = 983.

Problem 80 Square root digital expansion (19,014)

Problem: It is well known that if the square root of a natural number is not an integer, then it is irrational. The decimal expansion of such square roots is infinite without any repeating pattern at all.

The square root of two is 1.41421356237309504880…, and the digital sum of the first one hundred decimal digits is 475.

For the first one hundred natural numbers, find the total of the digital sums of the first one hundred decimal digits for all the irrational square roots.

Analysis: Notice that the 100 digits include digits to the right and left of the decimal. To make sure the answer is correct, it is necessary to calculate more digits than required to avoid rounding errors.

Solution: First need to exclude perfect squares. Two methods localcontext and Decimal from decimal standard library are quite useful for this problem. Use localcontext to set the precision as 102 (get this number because the square root of 99 has 102 digits with the last three digits as 272. if not selecting precision as 102, the last digit would be rounded to 3 instead of 2). Use Decimal(x).sqrt() to get the 102 digits of the square root of number x. Remove the decimal point and sum up the digits, then add all digit sums together to get the answer.

Function:

def sqr_sum(n):

    sums = 0
    for x in range(2, n+1):
        if int(x**0.5)**2 != x:
            with localcontext() as ctx:
                ctx.prec = 102
                root = str(Decimal(x).sqrt()).replace(".", "")
                sums += sum([int(root[j]) for j in range(100)])
    return sums

Answer: By running the function sqr_sum(100), the answer for the question is 40886.
